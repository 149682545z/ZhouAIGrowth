此技能指导创建独特的、生产级的前端界面，避免通用的“AI 废话”美学。通过对审美细节和创意选择的非凡关注，实现真正的运行代码。

用户提供前端需求：要构建的组件、页面、应用程序或界面。他们可能包含有关目的、受众或技术约束的上下文。

## 设计思维

在编码之前，了解上下文并致力于大胆的审美方向：
- **目的**：此界面解决什么问题？谁在使用它？
- **基调**：选择一个极端：极其极简、极大主义的混乱、复古未来主义、有机/自然、豪华/精致、俏皮/玩具感、社论/杂志感、粗野主义/原始、艺术装饰/几何、柔和/粉彩、工业/实用等。有很多风格可供选择。使用这些作为灵感，但要设计一个真实符合审美方向的作品。
- **约束**：技术要求（框架、性能、无障碍性）。
- **差异化**：是什么让它令人难忘？人们会记住的那一个点是什么？

**关键**：选择一个明确的概念方向并精准执行。大胆的极大主义和精致的极简主义都有效——关键是意图，而不是强度。

然后实现运行代码（HTML/CSS/JS、React、Vue 等），要求：
- 生产级且功能齐全
- 视觉冲击力强且令人难忘
- 具有清晰审美视角的内聚性
- 在每个细节上都经过精雕细琢

## 前端美学指南

专注于：
- **排版**：选择美观、独特且有趣的字体。避免使用 Arial 和 Inter 等通用字体；相反，选择能提升前端审美的独特选择；出人意料、富有性格的字体选择。将独特的展示字体与精致的正文字体配对。
- **色彩与主题**：致力于一种内聚的审美。使用 CSS 变量保持一致性。具有强烈视觉冲击力的主色调搭配鲜明的强调色，优于平淡、均匀分布的配色方案。
- **动效**：使用动画来实现交互效果和微交互。对于 HTML，优先考虑纯 CSS 解决方案。如果可用，对 React 使用 Motion 库。专注于高影响力的时刻：一个精心安排、带有交错显现（animation-delay）的页面加载比分散的微交互更能创造愉悦感。使用能带来惊喜的滚动触发和悬停状态。
- **空间构图**：出人意料的布局。非对称。重叠。对角线流动。打破网格的元素。慷慨的负空间或受控的密度。
- **背景与视觉细节**：营造氛围和深度，而不是默认使用纯色。添加与整体审美匹配的上下文效果和纹理。应用创意形式，如渐变网格、噪点纹理、几何图案、分层透明度、极具张力的阴影、装饰性边框、自定义光标和颗粒叠加。

切勿使用通用的 AI 生成美学，例如过度使用的字体族（Inter、Roboto、Arial、系统字体）、陈词滥调的配色方案（特别是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏特定上下文性格的模版化设计。

进行创造性的解释，做出在特定语境下感觉真正经过设计的意外选择。没有一个设计应该是相同的。在明暗主题、不同字体、不同审美之间变换。切勿在迭代中趋向于共同的选择（例如 Space Grotesk）。

**重要**：将实现复杂度与审美愿景相匹配。极大主义设计需要精细的代码、广泛的动画和效果。极简或精致的设计需要克制、精准，并仔细注意间距、排版和微妙的细节。优雅源于对愿景的良好执行。
