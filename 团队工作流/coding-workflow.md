# Coding Workflow with AI

为了不仅让自己变快，更让团队变强，我们将 AI 引入到软件开发的各个生命周期中。

## 1. 需求分析与设计
在写第一行代码之前，先用 AI 帮你是你理清思路。
- **User Story 生成**：输入简短的需求描述，让 AI 生成标准的“作为一名用户，我希望...”格式的故事，并列出验收标准 (Acceptance Criteria)。
- **架构设计辅助**：
    - 询问：“对于一个高并发的抢票系统，应该选用什么样的数据库架构？”
    - 让 AI 评审你的数据库设计，指出潜在的瓶颈。

## 2. 编码阶段 (Coding)
不要手写重复的劳动。
- **生成 Boilerplate**：
    - “生成一个基于 FastAPI 的用户注册接口，包含 Pydantic 验证和 SQLAlchemy 模型。”
    - “写一个 React 组件，展示用户列表，支持分页和搜索。”
- **正则与 SQL**：
    - “写一个正则表达式，匹配中国大陆手机号。”
    - “写一个 SQL 查询，找出过去30天活跃度最高的前10名用户。”
- **编写单元测试**：
    - 选中函数，直接命令：“为这个函数编写单元测试，覆盖边界情况。”

## 3. 代码审查 (Code Review)
AI 是你随叫随到的资深同事。
- **自我审查**：在提交代码前，先问 AI：“这段代码有什么潜在的 Bug 或性能问题？”
- **解释代码**：遇到看不懂的同事代码（或自己三个月前写的），让 AI “逐行解释这段代码的逻辑”。

## 4. 文档编写
没时间写文档？AI 有时间。
- **接口文档**：输入代码，让 AI 生成 Swagger/OpenAPI 格式的注释。
- **README**：让 AI 总结项目的功能、安装步骤和使用方法。

## 5. Debugging
别再对着报错发呆了。
- **复制报错信息**：直接把 Error Log 扔给 AI，它通常能一针见血地指出问题。
- **提供上下文**：记得把报错的代码段也一起发给它。
