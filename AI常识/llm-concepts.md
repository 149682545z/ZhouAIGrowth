# AI 核心概念 (Key Concepts)

要高效使用 AI，首先需要理解它的基本工作原理。不用担心，我们不需要深入数学公式，只需要掌握几个关键概念。

## 1. 大语言模型 (LLM)
**Large Language Model**，即大语言模型。你可以把它想象成一个阅读了互联网上几乎所有文本的“超级大脑”。它通过预测下一个词来生成文本。
- **示例**：GPT-4, Claude 3.5, DeepSeek-V3, Llama 3。
- **能力**：文本生成、翻译、摘要、逻辑推理、代码编写。
- **局限**：它没有真正的“意识”或“理解”，本质上是概率预测。

## 2. Token (词元)
模型处理文本的最小单位不是单词或字符，而是 Token。
- **如何计算**：
    - 英文：1个 Token ≈ 0.75 个单词（或者说 100个 Token ≈ 75 个单词）。
    - 中文：1个 Token ≈ 0.5 ~ 0.8 个汉字（取决于不同模型的切分方式）。
- **费用**：大多数 API 是按 Token 数量收费的（输入+输出）。

## 3. 上下文窗口 (Context Window)
这是 AI 的“短期记忆”容量。
- **含义**：模型在一次对话中能记住的 Token 总量（包括你输入的 Prompt 和它生成的回答）。
- **超出限制**：如果对话过长，最早的信息会被“遗忘”，导致 AI 无法联系前文。
- **现状**：
    - 早期模型（如 GPT-3.5）：4k ~ 16k tokens。
    - 现代模型（如 Claude 3.5, GPT-4o）：128k ~ 200k tokens（甚至更多），这允许你上传整本书或大量代码文件让它分析。

## 4. Temperature (温度)
控制模型输出随机性的一个参数，通常在 0 到 1 之间（有些模型可更高）。
- **低温度 (0.1 - 0.3)**：
    - 输出更确定、重复、保守。
    - **适用场景**：代码编写、数学解题、事实问答（因为你希望答案是唯一且正确的）。
- **高温度 (0.7 - 1.0)**：
    - 输出更随机、更有创意。
    - **适用场景**：创意写作、头脑风暴、聊天机器人。

## 5. 常见问题
### 幻觉 (Hallucination)
模型有时会自信地胡编乱造，尤其是在处理冷门知识、引用文献或它不知道答案的时候。
- **对策**：
    1. 要求模型“如果不知道由于请直接告诉我”。
    2. 让模型提供引用来源（如果它能联网）。
    3. 人工核查关键事实。

### 知识截止日期 (Knowledge Cutoff)
模型的知识是静态的，来自于它的训练数据。如果不联网，它不知道昨天发生了什么。
- **DeepSeek V3/R1** 等模型通常包含较新的数据，但仍有截止日期。
