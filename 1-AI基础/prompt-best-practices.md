# 提示词最佳实践 (Best Practices)

写好提示词 (Prompt) 是一门技艺。遵循以下原则，你可以显著提高 AI 输出的质量。

## 1. 核心原则：CRISPE 框架
虽然有很多框架，但核心思想通用的。我们可以参考 CRISPE 框架：
- **C**apacity and Role (能力与角色)：你希望 AI 扮演什么角色？
- **R**ole (角色)：同上，强调设定。
- **I**nsight (洞察/背景)：任务的背景信息是什么？
- **S**tatement (指令)：你具体想让 AI 做什么？
- **P**ersonality (个性)：回答的语气风格（如：专业、幽默、简洁）。
- **E**xperiment (实验)：要求 AI 提供多个选项或逐步改进。

## 2. 具体建议

### 🎯 清晰明确 (Be Specific)
不要让 AI 猜你的心思。
- ❌ **坏提示**：写个关于 Python 的代码。
- ✅ **好提示**：请写一个 Python 脚本，使用 `requests` 库抓取 `example.com` 的标题，并处理可能出现的网络异常。

### 🌍 提供上下文 (Provide Context)
给 AI 足够的信息，让它“懂你”。
- ❌ **坏提示**：解释一下这个错误。
- ✅ **好提示**：我是 Python 初学者，在这个代码段（粘贴代码）运行时遇到了 `IndexError`，请解释原因并告诉我如何修复。

### 🚧 设定约束 (Set Constraints)
告诉 AI **什么不要做**，或者**输出的格式**。
- **格式约束**：请以 Markdown 表格形式输出。
- **长度约束**：请将摘要控制在 200 字以内。
- **内容约束**：不要使用复杂的专业术语，假设我是五岁小孩。

### 🔄 迭代优化 (Iterate)
这一步至关重要。很少有 Prompt 是第一遍就完美的。
1. **第一次尝试**：通过上述原则写出 Prompt。
2. **评估结果**：看看 AI 哪里做得不好。
3. **修正指令**：针对不足之处，增加限定条件或更正背景信息。
4. **重复**：直到结果满意。

## 3. 结构化提示词示例
```markdown
# Role
你是一位资深的前端架构师。

# Context
我们需要重构现有的 Vue 2 项目到 Vue 3。

# Task
请列出重构的主要步骤和风险点。

# Format
请使用 Markdown 无序列表，重点加粗。
```
结构化的 Prompt 更容易被模型理解，也更容易维护。
